# 04 — خارطة الطريق (12 أسبوعاً)

كل مرحلة لها **مخرَج قابل للعرض** و**معيار قبول**. لا ننتقل قبل تحقق المعيار.

## المرحلة 0 — التأسيس (الأسبوع 1)
- إنشاء repo باسم `al-miqyas` تحت حساب GitHub المملوك للأمد
- مشروع Supabase جديد (منفصل عن vault و cards) + مشروع Vercel
- ربط `miqyas.al-amad.com.sa` (سجل CNAME → Vercel)
- تهيئة Next.js + TypeScript + Tailwind (RTL) + دليل ألوان الأمد
- **معيار القبول**: صفحة "قريباً" بالعربية تعمل على الدومين من iPhone/Safari

## المرحلة 1 — التينانت والهوية (الأسبوعان 2–3)
- تنفيذ `03-database-schema.sql` + اختبارات RLS آلية
- Supabase Auth (بريد + OTP) + شاشة دخول عربية
- إدارة الجهات: إنشاء جهة، شعار، لون، أعضاء وأدوار
- **معيار القبول**: مستخدمان في جهتين مختلفتين — كلٌّ يرى بيانات جهته فقط
  (اختبار تسرّب فعلي، ليس نظرياً)

## المرحلة 2 — المتدرّبون والمعرّف الموحّد (الأسبوعان 4–5)
- تسجيل متدرّب → `generate_trainee_code()` → بطاقة QR (تكامل مع نمط cards القائم)
- محتوى QR = رابط prefill للنموذج القبلي للبرنامج
- شاشة قائمة المتدرّبين + بحث بالكود
- **معيار القبول**: مسح QR بجوال حقيقي يفتح Jotform والحقل مملوء ومقفل

## المرحلة 3 — استيعاب Jotform (الأسبوعان 6–7)
- Edge Function `ingest-jotform`: webhook + توقيع سرّي
- مهمة reconciliation دورية عبر Jotform API (كل ساعة)
- محرك التصحيح `scoring.ts` مقابل `answer_key` للبرنامج
- ترحيل البيانات التاريخية: استيراد نتائج نبض الأمد القائمة (12 قبلي + 7 بعدي)
- **معيار القبول**: إرسال نموذج تجريبي يظهر في Supabase مصحَّحاً خلال < 10 ثوانٍ

## المرحلة 4 — الأداء اللحظي (الأسبوعان 8–9)
- `/api/xapi` باستقبال statements + مفتاح لكل جهة + rate limiting
- تعريف قاموس الأفعال/الكائنات المعتمد (بالتنسيق مع معمار AmadXR)
- شاشة الجلسة الحية (live view) — مرجعها البصري المحطة 2 في النموذج v2
- **معيار القبول**: لوحة الأمد ترسل جلسة كاملة وتظهر أحداثها حية باسم AMD-XXXXX

## المرحلة 5 — محرك الأثر (الأسبوع 10)
- `impact.ts`: فردي (قبلي/بعدي/ثقة/لحظي/حكم) + جماعي (متوسط، **أدنى درجة،
  انحراف معياري**، نسبة اجتياز)
- شاشة تقرير الأثر الفردي + الجماعي — العنوان الجماعي: رفع الأضعف وضغط التشتّت
- **معيار القبول**: تقرير دفعة نبض الأمد الحقيقية يُظهر: min 5→8، SD 1.62→0.79

## المرحلة 6 — الشهادات والتحقق (الأسبوع 11)
- إصدار شهادة عند تحقق شروط الاجتياز + QR يشير إلى `/verify/{code}`
- صفحة التحقق العامة (بلا دخول) عبر RPC `verify_certificate`
- قالب شهادة بشعار الجهة + هوية الأمد كمشغّل
- **معيار القبول**: مسح QR شهادة مطبوعة يعرض صفحة تحقق صحيحة على Safari

## المرحلة 7 — التجربة الميدانية والتقسية (الأسبوع 12)
- دورة نبض الأمد قادمة تُدار بالكامل عبر المنظومة (pilot)
- قياس نسبة المطابقة الفعلية (المستهدف ≥ 98%)
- توثيق تشغيلي: دليل الجهة + دليل المشرف الميداني
- **معيار القبول**: الدورة تكتمل قبلي→لحظي→بعدي→تقرير→شهادات دون تدخل يدوي
